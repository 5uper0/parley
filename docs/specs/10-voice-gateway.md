# voice/gateway.py — Spec

## Purpose
The gateway is an OpenAI-compatible `POST /v1/chat/completions` (SSE) server that ElevenLabs'
Custom-LLM mode calls and speaks verbatim — so **Parley itself is registered as the model**. Every
word the phone agent utters is generated by our code, never by a free-running LLM: the counterparty's
last turn is pinned to a concrete option, that option is run through the owner's private
`PreferenceSheet` (`sheet.evaluate`), and only the masked line chosen by `voice/policy.reply_for` is
streamed back. This is how masking/non-betrayal survive on the speech path — a commitment ("I accept")
can only be spoken *after* the sheet says the option is feasible, a refusal names no constraint, and
an unpinnable turn fails closed to a clarify. The sheet is loaded once into `_Gateway` and never
leaves the process. Zero new deps: stdlib `http.server` + SSE only.

## Public API

- `extract_option(messages: list) -> dict | None`
  — reads the **last `user` message** and pins it to a concrete option dict, or returns `None`
  (fail-closed). Accepts an embedded JSON object first (`re.search(r"\{.*\}", ...)`), else
  `key=value` tokens (`re.findall(r"([A-Za-z_][\w]*)\s*=\s*([^\s,;]+)")`) with values coerced.
  Non-pinnable natural language → `None`.
- `_coerce(v: str)` — `"true"/"false"` → bool, then `int`, then `float`, else the raw string.
- `class _Gateway.__init__(self, recipe_path: str, owner: str)` — loads the `DecisionSpec`, selects
  the party whose `owner` matches (raises `ValueError` if none), stores `self.owner`,
  `self.sheet = party.to_sheet()`, `self.options = spec.options` (public), and
  `self.use_brain = bool(os.environ.get("OPENROUTER_API_KEY"))`.
- `_Gateway.decide_reply(self, messages: list) -> str` — the per-turn decision. With a key:
  `voice.brain.respond(messages, self.sheet, options_hint=self.options)`. Without: `extract_option`
  → `self.sheet.evaluate(option)` (or `None`) → `voice.policy.reply_for(ev)`.
- `make_handler(gateway: "_Gateway", bearer: str | None)` — returns a `BaseHTTPRequestHandler`
  subclass bound to that gateway and bearer.
- `serve(recipe_path, owner, bearer, host="127.0.0.1", port=8013) -> ThreadingHTTPServer`
  — builds the gateway and a `ThreadingHTTPServer`; caller runs `.serve_forever()`.

### HTTP endpoint
- **Method / path:** `POST /v1/chat/completions` (trailing slash tolerated via `rstrip("/")`; any
  other path → `404`). Only POST is handled.
- **Auth:** if `bearer` is set, `Authorization` header must equal `Bearer {bearer}` exactly, else
  `401 {"error":"unauthorized"}`. If `bearer` is `None`, auth is skipped.
- **Body bound:** `Content-Length` must be `> 0` and `<= MAX_BODY (64 * 1024)`, else `413`.
- **Request JSON:** `{"messages": [ {"role","content"}, ... ], ...}` (OpenAI shape; ElevenLabs also
  sends `stream`). `messages` must be a list, else `400`.
- **SSE response:** `200`, `Content-Type: text/event-stream`, `Cache-Control: no-cache`. Frames from
  `_sse_chunks(reply)`:
  1. `data: {"id":"chatcmpl-parley-<ms>","object":"chat.completion.chunk","model":"parley-voice","choices":[{"index":0,"delta":{"role":"assistant","content":<reply>},"finish_reason":null}]}`
  2. `data: {... "choices":[{"index":0,"delta":{},"finish_reason":"stop"}]}`
  3. `data: [DONE]`
  Each frame is `"data: " + json.dumps(obj) + "\n\n"`.

## Data model
- **Option** (`dict`): the concrete attributes on the table, e.g. `{"heir1":50,"heir2":50,"legal_ok":true}`.
  Produced by `extract_option`; the same shape `sheet.evaluate` and `Constraint.check` consume.
- **SSE frame:** OpenAI `chat.completion.chunk`; the whole reply ships as a single content delta in
  frame 1 (no token-by-token streaming), then a stop frame, then `[DONE]`.
- **Reply string:** always one of `voice.policy.ACCEPT` / `REFUSE` / `CLARIFY` in the deterministic
  path, or the brain's spoken line in the keyed path.

## Invariants (must never break)
- **A commitment is spoken only after `sheet.evaluate`.** `decide_reply` never emits `ACCEPT` except
  via `reply_for(ev)` where `ev = self.sheet.evaluate(option)` and `ev.feasible` is true (deterministic
  path), or via the brain, which can only say "accept" after a `consider_option` tool result of
  `acceptable=true` (also `sheet.evaluate`). There is no branch that agrees without the sheet.
- **Refusals are masked.** The refusal line is `voice.policy.REFUSE`, which names no attribute, no
  threshold, no constraint, no score. `_Gateway` never returns `ev.violated` or `ev.score` to the wire.
- **The sheet never leaves the process.** `self.sheet` is built from the recipe and only ever consumed
  by `evaluate`/`brain.respond` locally; only `self.options` (public) is passed as `options_hint`.
- **Fail-closed when the option can't be pinned.** `extract_option` returning `None` → `ev = None` →
  `reply_for(None)` → `CLARIFY`. The agent asks to restate rather than inventing agreement.
- **The body is bounded** (`MAX_BODY = 64 * 1024`, same discipline as `parley/net/bot.py`) and auth is
  enforced before the body is read.

## Interactions
- **parley core:** `from parley.spec import DecisionSpec` (`.load`, `.parties`, `PartySpec.to_sheet`,
  `.options`); `sheet.evaluate(option) -> Evaluation` from `parley/preferences.py`.
- **voice/policy.py:** `reply_for(ev)` maps the `Evaluation` (or `None`) to a whitelisted line.
- **voice/brain.py:** imported lazily inside `decide_reply` only when `OPENROUTER_API_KEY` is set;
  receives `messages`, `self.sheet`, `options_hint=self.options` — never the sheet in a prompt.
- **ElevenLabs (external):** speaks whatever the SSE stream returns; sends `PARLEY_GATEWAY_BEARER` as
  the Custom-LLM Bearer. Crosses the boundary in: OpenAI-shaped `messages` in, SSE `chat.completion.chunk`
  frames out. No sheet, no `violated`, no `score` ever crosses.

## Failure modes / edge cases
- **Unparsable speech** (no JSON, no `key=value`): `extract_option` → `None` → `CLARIFY` (fail-closed).
  Covered by `test_unparsable_turn_fails_closed_to_clarify`.
- **Missing bearer / wrong token** (when `bearer` set): `401 {"error":"unauthorized"}`. Covered by
  `test_missing_bearer_is_unauthorized`.
- **Missing OpenRouter key:** `use_brain` is false → deterministic `extract_option` path. This is the
  default for tests and key-less local runs (README: "otherwise the gateway falls back to a fail-closed
  structured extractor").
- **Bad/oversized body:** `Content-Length <= 0` or `> MAX_BODY` → `413`; non-JSON or missing/non-list
  `messages` (`ValueError | KeyError | AssertionError`) → `400`.
- **Wrong path / method:** non-`/v1/chat/completions` POST → `404`; no GET handler.
- **Owner not in recipe:** `_Gateway.__init__` raises `ValueError` listing the recipe's parties (fails
  at startup, not per request).

## Test coverage
`tests/test_voice_gateway.py` (starts a real server on port 0, emulates ElevenLabs, reads SSE):
- `test_feasible_proposal_is_accepted` — `{"heir1":50,"heir2":50,"legal_ok":true}` → `ACCEPT`.
- `test_red_line_proposal_is_refused_and_masked` — `heir2=10` (crosses `heir2 >= 30`) → `REFUSE`,
  and asserts `"heir2"`, `"30"`, `"red"` do **not** appear in the reply.
- `test_key_value_speech_also_parses` — `"let's do heir1=50 heir2=50 legal_ok=true"` → `ACCEPT`
  (the `key=value` extractor branch).
- `test_unparsable_turn_fails_closed_to_clarify` — chit-chat with no option → `CLARIFY`.
- `test_missing_bearer_is_unauthorized` — server started with a bearer, request without one → HTTP 401.

## Open questions / roadmap
- **Claude-backed NL extractor.** The module docstring notes the structured `extract_option` is the
  MVP; a Claude extractor (NL → option) "drops in behind an optional extra later" keeping the
  `str -> dict|None`, fail-closed contract. In the current code the LLM path is `voice/brain.py`
  (tool-based), not an extractor swap.
- **True token streaming.** Today the whole reply ships as one delta; ElevenLabs speaks it fine, but
  incremental streaming would lower spoken latency.
- **Telephony wiring** (README "Wiring the phone"): cloudflared/ngrok tunnel + ElevenLabs Custom-LLM
  URL + Zadarma SIP trunk, then the by-voice red-team gate (normal deal / red-line push /
  prompt-injection) before any self-call — operator-only, not yet in code.
- Voice is an **optional I/O layer around the unchanged core**; it is not itself a numbered ROADMAP
  experiment (Exp 1 is `parley/elicit.py`). Voice input (mic → STT → `elicit`) is flagged as the
  cheapest first probe that can't break an invariant.
